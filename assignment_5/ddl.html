DATA DEFINITION LANGUAGE (DDL)

CREATE

The CREATE command is used to define a new database object such as a table, index, view, or stored procedure. This command specifies the structure, columns, data types, constraints, and other properties for the object being created.

DROP

The DROP command is used to delete an existing database object such as a table, index, view, or stored procedure. This command removes the object and all its data from the database, making it irrecoverable.

ALTER

The ALTER command is used to modify the structure of an existing database object such as a table, index, or view. This command allows you to add, delete, or modify columns, change data types, and apply other structural changes to the object.

ADD

The ADD command is used with the ALTER statement to add a new column or constraint to an existing table or database object.

UPDATE

The UPDATE command is used to modify existing data in a table. This command allows you to change the values of one or more columns for one or more rows based on a specified condition.

SET

The SET command is used to assign a value to a variable or to specify the value of a column in an UPDATE statement. It is commonly used to modify the values of columns in a table or to define user variables.

INSERT

The INSERT command is used to add new rows of data into a table. This command specifies the table, the columns, and the values to be inserted.

SELECT

The SELECT command is used to retrieve data from one or more tables. This command specifies the columns to be retrieved and can include conditions, sorting, and grouping to filter and organize the results.

By understanding and using these SQL commands, you can effectively manage and manipulate data within a database, ensuring data integrity, consistency, and efficient retrieval of information.

TRUNCATE

The TRUNCATE command removes all records from a table but retains the table structure for future use. This is different from DROP, which removes the entire table structure.

RENAME

The RENAME command is used to change the name of a database object. This is useful for reorganizing database objects without affecting their structure or data.

COMMENT

The COMMENT command adds comments to the data dictionary. Comments can be added to tables and columns to provide descriptions and other useful information about the database objects.



By understanding and using these SQL commands, you can effectively manage and manipulate data within a database, ensuring data integrity, consistency, and efficient retrieval of information.
